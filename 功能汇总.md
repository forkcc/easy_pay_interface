## 五、Telegram Bot（cmd/bot）

- **用途**：商户群、渠道群、超级用户；业务或运维通过 HTTP 向指定商户群/渠道群推送消息。
- **环境变量**：
  - `BOT_TOKEN`：Telegram Bot Token（未设置时仅起 HTTP 服务，不跑机器人）。
  - `BOT_SUPER_ACCOUNTS`：超级用户 Telegram 用户名，逗号分隔（不含 @，大小写不敏感；仅超级用户可绑定群、私聊广播）。
  - `BOT_HTTP_ADDR`：可选，默认 `:5670`。
- **数据库**：需执行迁移 `migrations/005_bot_chat_bind.sql`，表 `bot_chat_bind`（chat_id、bind_type：merchant/channel、bind_id：商户 ID 或 interface_code）。
- **群内命令**（仅超级用户可执行「绑定」）：
  - `绑定商户 <商户ID>`：将本群设为该商户的商户群。
  - `绑定渠道 <接口编码>`：将本群设为该支付接口的渠道群。
  - `绑定信息` / `/who`：查看本群当前绑定。
  - `余额` / `cx` / `/balance`：商户群查余额，渠道群提示用管理端。
  - `帮助` / `/help`：命令说明。
- **私聊**：超级用户发送的私聊消息会广播到所有已绑定群。
- **HTTP 推送**（供业务/运维调用）：
  - `POST /transfer/merchant`  
    Body: `{"merchantId": 1, "message": "通知内容"}`  
    向该商户绑定的群推送消息；未绑定或发送失败返回 404。
  - `POST /transfer/channel`  
    Body: `{"interfaceCode": "WXPAY", "message": "通知内容"}`  
    向该接口编码绑定的渠道群推送消息；未绑定或发送失败返回 404。
  - `GET /health`：返回 200 ok。
